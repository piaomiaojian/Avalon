阿瓦隆遊戲 - 各畫面跳轉流程
================================

1. 主選單 (mainMenu)
===================
進入條件：頁面載入完成
顯示內容：
- 創建房間按鈕
- 加入房間按鈕

跳轉邏輯：
- 點擊「創建房間」→ 房間區域 (roomArea)
- 點擊「加入房間」→ 掃描區域 (scanContainer)

2. 房間區域 (roomArea)
=====================
進入條件：
- 房主：創建房間後
- 玩家：WebRTC連接建立後

顯示內容：
- 房間狀態
- 房主控制按鈕（僅房主可見）
  - 加入玩家按鈕
  - 開始遊戲按鈕（僅在支援人數時顯示）
- 房間成員列表
- 房間聊天區域

跳轉邏輯：
- 房主點擊「加入玩家」→ QR碼區域 (qrContainer)
- 房主點擊「開始遊戲」→ 遊戲操作區域 (gameOperationArea)
- 所有玩家：連接建立後自動進入

3. QR碼區域 (qrContainer)
========================
進入條件：
- 房主：點擊「加入玩家」後（顯示offer信號）
- 加入者：掃描房主QR碼後（顯示answer信號）

顯示內容：
- QR碼標題
- QR碼圖片
- QR碼文字內容
- 按鈕配置：
  - 房主：返回按鈕 + 掃描按鈕
  - 加入者：返回按鈕

跳轉邏輯：
- 房主：點擊「返回房間」→ 房間區域
- 房主：點擊「掃描」→ 掃描區域
- 加入者：連接建立後自動進入房間區域

4. 掃描區域 (scanContainer)
==========================
進入條件：
- 房主：從QR碼區域點擊「掃描」後
- 加入者：點擊「加入房間」後

顯示內容：
- 掃描標題
- 相機預覽
- 掃描框架和指示器
- 手動輸入欄位
- 按鈕配置：
  - 房主：重新掃描按鈕 + 返回按鈕
  - 加入者：重新掃描按鈕 + 返回按鈕

跳轉邏輯：
- 房主：掃描成功後自動進入房間區域
- 加入者：掃描成功後顯示QR碼區域
- 房主：點擊「返回」→ QR碼區域
- 加入者：點擊「返回」→ 主選單

5. 遊戲操作區域 (gameOperationArea)
==================================
進入條件：
- 房主點擊「開始遊戲」後
- 角色分配完成後

顯示內容：
- 遊戲標題
- 遊戲階段
- 玩家頭像（左右兩側）
- 遊戲訊息（中間）
- 查看遊戲按鈕

跳轉邏輯：
- 點擊「查看遊戲」→ 遊戲查看區域 (gameViewArea)

6. 遊戲查看區域 (gameViewArea)
==============================
進入條件：
- 從遊戲操作區域點擊「查看遊戲」

顯示內容：
- 遊戲統計標題
- 任務進度統計
- 投票記錄統計
- 玩家狀態統計
- 返回遊戲按鈕

跳轉邏輯：
- 點擊「返回遊戲」→ 遊戲操作區域 (gameOperationArea)

7. 遊戲區域 (gameArea) - 舊版，已棄用
=====================================
進入條件：無（已棄用）
顯示內容：舊版遊戲介面
跳轉邏輯：無

特殊跳轉規則：
==============
1. 連接建立後：
   - 房主：保持在房間區域
   - 加入者：進入房間區域

2. 錯誤處理：
   - 掃描失敗：顯示錯誤訊息，可重試
   - 連接失敗：返回主選單
   - 人數不支援：隱藏開始遊戲按鈕

3. 隱藏邏輯：
   - 所有區域使用 .hidden class 控制顯示
   - 使用 hideAllAreas() 方法統一管理
   - 使用 showElement() 和 hideElement() 方法切換
